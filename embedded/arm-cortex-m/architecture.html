// 02 — REGISTER SET (32-BIT EACH)
R0a1
Tham số 1 / Giá trị trả về
Argument 1 khi gọi hàm. Kết quả return nằm ở đây. Caller-saved — có thể bị thay đổi sau khi gọi hàm.
R1a2
Tham số 2
Argument 2. Cùng nhóm caller-saved với R0, R2, R3.
R2a3
Tham số 3
Argument 3. Hàm có >4 tham số → phần còn lại đẩy lên stack.
R3a4
Tham số 4
Argument 4. R0–R3 bị clobber qua function call — compiler tự lưu nếu cần.
R4
Callee-saved #1
Nếu hàm dùng R4 → phải PUSH R4 lúc đầu, POP R4 trước return. FreeRTOS lưu R4–R11 khi context switch.
R5
Callee-saved #2
Bảo toàn qua function calls. Dùng cho biến local dài hạn trong hàm.
R6
Callee-saved #3
Bảo toàn qua function calls.
R7
Callee-saved #4 / Frame Pointer
Đôi khi GCC dùng làm Frame Pointer (FP) để debug stack frame.
R8
Callee-saved #5
Bảo toàn qua function calls.
R9
Callee-saved #6
Đôi khi dùng làm Static Base (SB) pointer.
R10
Callee-saved #7
Bảo toàn qua function calls.
R11
Callee-saved #8
R4–R11 là nhóm FreeRTOS phải lưu/khôi phục thủ công khi context switch (stacking phần mềm).
R12IP
Intra-procedure Scratch
Dùng tạm bởi linker veneers (glue code). Caller-saved.
R13SP
Stack Pointer ★★★
Có 2 bank: MSP (Main — dùng cho interrupt/kernel) và PSP (Process — FreeRTOS cấp riêng cho từng task). Luôn aligned 8 bytes.
R14LR
Link Register ★★
Tự động lưu địa chỉ return khi gọi BL/BLX. Khi exception: LR = EXC_RETURN (giá trị đặc biệt báo cách return từ interrupt).
R15PC
Program Counter ★★★
Luôn trỏ đến lệnh sắp fetch (hiện tại + 4). Context switch = thay đổi PC + SP để nhảy sang task khác. Bit 0 luôn = 0 (Thumb mode).
★ FreeRTOS Context Switch: Hardware tự lưu R0–R3, R12, LR, PC, xPSR (8 regs). FreeRTOS tự lưu R4–R11 (8 regs). Tổng = 16 registers × 4 bytes = 64 bytes mỗi lần switch.

// CPU cập nhật xPSR sau mỗi phép tính — if/else dùng flags này
int a = 5 - 5;   // Z=1 (bằng 0), N=0, C=1
int b = -3;      // N=1 (âm)
int c = 200 + 100; // uint8: C=1 (carry out)

// Lệnh branch dùng flags:
if (a == 0) { }  // CPU: BEQ → check Z flag
if (b <  0) { }  // CPU: BMI → check N flag
if (a != b)  { }  // CPU: BNE → check Z flag = 0
// 06 — PIPELINE 3-STAGE (CORTEX-M3/M4)
STAGE / CYCLE T=1T=2T=3 T=4T=5T=6T=7
FETCH (đọc lệnh) I1 I2 I3 I4 I5 I6 I7
DECODE (giải mã) — I1 I2 I3 I4 I5 I6
EXECUTE (thực thi) — — I1 I2 I3 I4 I5
⚡ Khi interrupt xảy ra tại T=4: CPU phải flush I5, I6, I7 đang trong pipeline → latency ~12 cycles. Đây là lý do FreeRTOS tính worst-case interrupt response time dựa vào đặc điểm này. Cortex-M7 có pipeline 6-stage nên latency cao hơn nhưng throughput tốt hơn nhiều.
// 07 — 3 EXCEPTIONS FREERTOS SỬ DỤNG
TICK
EXCEPTION #15
SysTick
Bộ đếm thời gian hệ thống. Mỗi 1ms (configTICK_RATE_HZ=1000) sẽ fire một lần.

• Tăng xTickCount
• Wake up tasks đang delay
• Kiểm tra nếu cần switch → trigger PendSV

Priority: Thấp nhất (configKERNEL_INTERRUPT_PRIORITY)
SWITCH
EXCEPTION #14
PendSV
Thực hiện context switch thực sự. Được thiết kế đặc biệt để có priority thấp nhất — chỉ chạy khi không có interrupt nào khác pending.

• Push R4–R11 của task hiện tại
• Lưu PSP
• Chọn task tiếp theo (scheduler)
• Khôi phục PSP + R4–R11 của task mới
START
EXCEPTION #11
SVC
SuperVisor Call — chỉ dùng 1 lần duy nhất khi vTaskStartScheduler() khởi động task đầu tiên.

• Chuyển từ privileged sang unprivileged
• Switch stack từ MSP sang PSP
• Jump vào task đầu tiên

Sau đó không dùng SVC nữa (PendSV đảm nhận)
// 08 — NVIC & PRIORITY RULES
Cortex-M4 có 4-bit priority = 16 mức (0 = cao nhất, 15 = thấp nhất).
configMAX_SYSCALL_INTERRUPT_PRIORITY = 5 (ví dụ)
PRIORITY GỌI FREERTOS API? VÍ DỤ SỬ DỤNG GHI CHÚ
0–4
✗ KHÔNG được Hard real-time ISR, motor control, safety-critical Chạy ngay lập tức, không bị FreeRTOS mask
5–14
✓ ĐƯỢC (dùng ...FromISR) UART RX, SPI, I2C, Button interrupt Dùng xQueueSendFromISR(), xSemaphoreGiveFromISR()
15
— (kernel only) SysTick, PendSV FreeRTOS tự đặt, thấp nhất để không block app IRQ 
// FreeRTOSConfig.h — cấu hình priority
#define configMAX_SYSCALL_INTERRUPT_PRIORITY   5
#define configKERNEL_INTERRUPT_PRIORITY        15

// Trong code ứng dụng:
NVIC_SetPriority(USART1_IRQn, 6);  // ✓ Được gọi FromISR()
NVIC_SetPriority(TIM1_IRQn,   2);  // ✗ KHÔNG gọi FreeRTOS API!
NVIC_EnableIRQ(USART1_IRQn);

// Trong ISR priority 6 — ĐÚNG:
void USART1_IRQHandler(void) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xQueueSendFromISR(rxQueue, &byte, &xHigherPriorityTaskWoken);
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken); // Trigger PendSV nếu cần
}
⚠️ Lỗi phổ biến nhất: gọi xQueueSend() (không có FromISR) bên trong ISR → crash ngay lập tức hoặc deadlock. Luôn dùng ...FromISR() variants bên trong interrupt handler!
