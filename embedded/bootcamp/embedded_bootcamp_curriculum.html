PROJECT Module 1 — Capstone Projects
  • P1.1
    Embedded Data Structure Library
    Viết một thư viện C hoàn chỉnh gồm: static-allocated linked list, ring buffer, min-heap — hoạt động không có dynamic memory (malloc-free), sử dụng được trên MCU không có OS.
  • P1.2
    Task Dispatcher với Priority Queue
    Xây dựng một task scheduler đơn giản dựa trên min-heap: add task với priority, pop task có priority cao nhất, benchmark thời gian thực thi.
  • P1.3
    Sensor Data Processing Pipeline
    Cài đặt pipeline xử lý dữ liệu cảm biến: sliding window buffer, moving average với circular queue, peak detection bằng BST. Benchmark memory footprint.

Programming
Phương pháp lập trình phản ứng sự kiện là DNA của các hệ thống nhúng phức tạp. Từ finite state machine đến active object pattern, module này dạy bạn thiết kế firmware có thể scale và maintain được.
3 Tuần học C/C++ 3 Mini Projects 1 Capstone
WEEK 04 Finite State Machines
Nội dung học
  • FSM concept: states, events, transitions, actions
  • FSM cài đặt bằng switch-case — anti-pattern phân tích
  • Table-driven FSM — data-driven approach
  • Hierarchical State Machine (HSM) — superstate, history
  • UML State Diagram — vẽ và đọc state chart
  • Debounce, button FSM thực chiến trên MCU
  • FSM cho giao thức truyền thông (UART frame parsing)
  • Guard conditions và extended states
WEEK 05 Event Queue & Message Passing
Nội dung học
  • Event-driven vs polling — khi nào dùng cái gì
  • Event Queue: FIFO, priority queue cho events
  • Event descriptor: type, data payload, source, timestamp
  • Publish-Subscribe pattern từ zero
  • Observer pattern trong C — callback registration
  • ISR-safe event posting — critical section management
  • Message pool — zero-copy event passing
  • Deferred interrupt processing pattern
WEEK 06 Active Objects & Frameworks
Nội dung học
  • Active Object pattern — thread + event queue + FSM
  • Asynchronous messaging giữa các active objects
  • Time events: one-shot timer, periodic timer
  • QP/C framework concepts — QActive, QMsm
  • Run-to-completion (RTC) semantics
  • Event-driven vs interrupt-driven architecture
  • Tracing & debugging event-driven systems
  • Design pattern: Command, Chain of Responsibility

  • P2.2
    Custom Event Framework (mini-EDF)
    Tự xây dựng mini event-driven framework: event pool, subscriber registry, dispatcher loop. Hỗ trợ priority events và wildcard subscription. Benchmark latency.

    Multi-Component Embedded Application
    Ứng dụng nhúng gồm 3 active objects giao tiếp qua event: sensor reader, data processor, output controller. Chạy trên STM32 hoặc Raspberry Pi Pico.

04 MODULE
Tuần 10 — Tuần 12  ·  RTOS Layer — Ultimate Challenge
Build Your
Own RTOS
Đỉnh cao của chương trình: tự tay code từng thành phần cốt lõi của một Real-Time Operating System. Từ context switching trên ARM Cortex-M đến semaphore, message queue, memory allocator — mọi thứ được xây từ zero. Sản phẩm cuối: AK-mOS chạy được trên phần cứng thực.
3 Tuần học ARM Cortex-M / C Iterative Build 1 Full RTOS
WEEK 10 Context Switch & Scheduler
Nội dung học
  • ARM Cortex-M architecture: registers, stack frame, PSP/MSP
  • SysTick timer — heartbeat của RTOS
  • PendSV exception — context switching mechanism
  • Task Control Block (TCB) structure
  • Stack initialization cho từng task
  • Round-robin scheduler từ scratch
  • Priority-based preemptive scheduler
  • Idle task và WFI (Wait For Interrupt) power saving
WEEK 11 Synchronization & IPC
Nội dung học
  • Critical section: disable/enable interrupt, BASEPRI
  • Binary semaphore — implementation từ struct
  • Counting semaphore — producer/consumer pattern
  • Mutex với priority inheritance — tránh priority inversion
  • Message queue — lock-free ring buffer
  • Task notification — lightweight signaling mechanism
  • Event flags / event group
  • Deadlock detection và tránh deadlock
WEEK 12 Memory, Timers & Polish
Nội dung học
  • Memory management: fixed-size block allocator (pool)
  • Heap_1 / Heap_4 style memory allocator implementation
  • Stack overflow detection — paint pattern, MPU guard
  • Software timer: one-shot và periodic, timer daemon task
  • Tick-less mode — dynamic tick cho low power
  • RTOS-aware debugging với SEGGER SystemView
  • CPU usage statistics, task runtime monitoring
  • Porting AK-mOS lên phần cứng thực + validation
PROJECT Module 4 — AK-mOS: Full RTOS Build
  • P4.1
    AK-mOS Core: Scheduler & Context Switch
    Implement AK-mOS kernel core: TCB, stack init, PendSV handler bằng assembly, SysTick ISR, round-robin + priority scheduler. Chạy được 4 tasks đồng thời trên STM32.
  • P4.2
    AK-mOS IPC: Semaphore, Mutex, Queue
    Thêm IPC primitives: binary & counting semaphore, mutex với priority inheritance, lock-free message queue. Test với classic problems: dining philosophers, producer-consumer.
  • P4.3
    AK-mOS Complete: Memory + Timers + Demo App
    Hoàn thiện AK-mOS: memory pool allocator, software timers, stack guard. Demo application: multi-task sensor system sử dụng toàn bộ IPC primitives. Viết documentation đầy đủ.
